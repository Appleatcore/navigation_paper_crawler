# 化学方向最小改动计划

目标是把当前仓库作为基线冻结，不在这个工作区直接改化学方向代码；后续在新的工作区里重新 `git clone`，再切一个专门的分支来做化学方向改造。这个文档只记录计划和事前准备，不执行代码修改。

## 1. 当前工作区策略

- 当前工作区保持不动，不继续在这里叠加化学方向改动。
- 当前工作区只作为参考基线，用来对照导航版本和化学版本的差异。
- 如果后续确认要实施修改，再在新的工作区中操作，避免污染现有整理结果。

## 2. 新工作区策略

- 在另一台机器或新的目录里重新 `git clone` 当前仓库。
- 在新工作区里创建一个专用分支，例如 `chemistry`、`chemistry-assistant` 或 `chem-paper-crawler`。
- 所有化学方向的改动只在这个新分支上完成，便于回滚和审查。
- 新工作区不要直接复用当前工作区的 `config.local.json`、日志文件和临时分析文件。

## 3. 事前准备

- 先确认化学方向的具体子领域，不要一开始就覆盖整个化学。
- 先确定检索范围来自哪些数据源：先保留 arXiv / Semantic Scholar，后续再决定是否补 OpenAlex、Crossref、PubMed 或 ChemRxiv。
- 先准备一份化学方向关键词表，最好分成核心词、扩展词和排除词。
- 先准备 Notion 数据库字段草案，确认最终要展示哪些列。
- 先确认 LLM 评分标准，是偏“前沿性”，还是偏“实验可信度/机理完整度/可复现性”。

## 4. 最小改动范围

- 关键词配置：把 navigation 关键词换成化学方向关键词。
- 过滤逻辑：把当前的导航过滤器换成化学领域过滤器。
- 标签与文案：把 Notion 标签、README、日志文本中的 navigation 改成 chemistry / chemistry frontier / chemistry literature 等。
- LLM prompt：把导航评分标准换成化学论文评分标准。
- 可选字段：保留现有 Notion 字段结构，先复用 `Name`、`Authors`、`Venue`、`Date`、`Abstract`、`DOI`、`PDF Link`、`Tags`、`Institutions`、`Recommend Score`、`Recommend Rationale`。

## 5. 最小实施顺序

- 第一步只改关键词和过滤器，先验证能不能稳定抓到化学论文。
- 第二步改 LLM prompt 和标签，让评分结果更符合化学学生需要。
- 第三步再考虑是否接入新的化学专用数据源。
- 第四步再根据样本结果微调字段和 Notion 展示。

## 6. 化学方向需要提前提供的数据

- 研究方向：例如有机合成、催化、电化学、材料化学、高分子、药物化学、计算化学、分析化学、化学生物学等。
- 关键词：核心关键词、扩展关键词、排除关键词。
- 期刊或来源偏好：例如 Nature Chemistry、JACS、Angew. Chem.、ACS Catalysis、Chemical Science 等。
- 文献类型偏好：综述、方法论文、机理论文、材料论文、应用论文。
- 评分偏好：更重视前沿性、可复现性、实验完整度，还是与课题的相关性。

## 7. 初步验收标准

- 新工作区能正常启动，不影响当前工作区。
- 能检索到化学方向论文，而不是继续抓 navigation 论文。
- 能把论文稳定写入 Notion。
- LLM 评分提示词已经切换到化学方向。
- 关键词、标签、文档都已经改成化学版本，不再混用 navigation 文案。

## 8. 风险点

- 化学论文来源比导航更分散，单靠 arXiv 可能覆盖不够。
- 如果关键词过宽，会混入材料、工程、药化等大量噪声。
- 如果关键词过窄，会漏掉前沿论文。
- 如果没有先定义好子领域，后面的评分和过滤会反复返工。

## 9. 当前状态

- 当前工作区不改代码。
- 等你检查并确认这份计划后，再在新工作区里实际实施。
